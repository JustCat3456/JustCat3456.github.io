# Design System: Retro 80s GUI Portfolio

## 1. Visual Theme & Atmosphere
80年代初頭のパーソナルコンピュータ（Classic Macintoshや初期OS）のグラフィカル・ユーザー・インターフェース（GUI）を現代的に再解釈した、レトロ・インダストリアルかつ温かみのあるインターフェース。
デスクトップ、重なり合うウィンドウ、ビットマップ風のテクスチャを活かしつつ、スムーズなスプリングアニメーションとアシンメトリーなレイアウトで現代的な洗練さを与えます。
雰囲気：**「温かみのあるヴィンテージ・オフィス・ワークステーション」**
密度：**6 (適度な情報密度と余白のバランス)**
分散（Variance）：**7 (アシンメトリーなウィンドウ配置とオフセットシャドウ)**
モーション：**5 (慣性のあるドラッグ＆ドロップとウィンドウ展開時のスプリング挙動)**

---

## 2. Color Palette & Roles
80年代のハードウェアとモノクロ画面から着想を得た、紙の温かみを感じるパレットです。

- **Warm Canvas** (`#F5F5F0`) — メインの背景。古いプラスチック筐体やマニュアル用紙のような温かみのある白。
- **System Paper** (`#EAEAE0`) — ウィンドウやカードの背景。階層を定義する薄いベージュグレー。
- **Charcoal Ink** (`#1C1A17`) — テキスト、ボーダー、主要なグラフィックス。純粋な黒（#000000）を避け、コントラストを保ちつつ目に優しい炭色。
- **Muted Dust** (`#7A7870`) — サブテキスト、補助的な枠線、不活性なステータス。
- **Classic Terminal Green** (`#2E8B57`) — 唯一のアクセントカラー。かつてのグリーンモニターを彷彿とさせる、彩度を抑えた深みのあるセージグリーン。リンク、アクティブ状態、プライマリボタン用。

> [!WARNING]
> 純粋な黒（#000000）や、サイバーパンク風の過剰に光るネオン（紫・青のネオン光）は一切使用しません。

---

## 3. Typography Rules
- **Display (大見出し):** `Satoshi-Bold` もしくは `Cabinet Grotesk`。文字間を少し詰め（tracking-tight）、自信に満ちたアシンメトリーな配置で使用。
- **Body (本文):** `Satoshi-Regular`。行間を十分に開け、1行あたり最大65文字（65ch）に制限。
- **Mono (システム/コード):** `JetBrains Mono` または `Courier`。ウィンドウのタイトルバー、メニュー項目、ステータス数値、コマンドラインターミナル部分に適用。
- **Banned:** `Inter` およびシステム標準の `Segoe UI` や `Arial` は使用禁止。

---

## 4. Component Stylings
- **GUI Window (ウィンドウ):**
  - 背景は `System Paper`。
  - 枠線は `Charcoal Ink`（1px〜2px）。
  - タイトルバーがあり、左側に「閉じるボタン（四角いマーク）」、右側にドット風の縮小/拡大パーツ。
  - 右下にはリサイズを暗示する斜め線のハッチング。
  - 影はぼかさず、`Charcoal Ink` を数ピクセル右下にずらしたソリッドなオフセットシャドウを使用。
- **Buttons (ボタン):**
  - 基本は四角形。太さ1pxの枠線。
  - アクティブ（クリック）時に右下に1px沈み込み、ソリッドな影が消えるタクタイル（触覚的）フィードバックをCSSで表現。
  - プライマリボタンは `Classic Terminal Green` の背景、文字は `Warm Canvas`。
- **Inputs (入力フォーム):**
  - ラベルは入力エリアの上に配置。
  - 入力エリアは白（`#FFFFFF`）背景に `Charcoal Ink` の枠線。フォーカス時は `Classic Terminal Green` の細いアウトライン。
- **Loaders (ローダー):**
  - ドット絵風のプログレスバー（`[████░░░░░░] 40%`）のスタイル。ローディングアニメーションは等幅ステップで進行。

---

## 5. Layout Principles
- **Window Stack Layout:**
  - 画面上に複数の「レトロOS風ウィンドウ」が重なり合って配置されているようなレイアウト。
  - ヒーローセクションは左右非対称（アシンメトリー）。左側に大きなテキストとインラインピクセルアイコン、右側にメインのGUIウィンドウ（自己紹介やステータス）を配置。
  - レスポンシブ設計：画面幅が768px未満の場合、ウィンドウは重なりを解消し、縦一列のシンプルなカードレイアウトに自動スタックします。
- **Grid Structure:**
  - CSS Grid を優先的に使用し、厳密なグリッド上にウィンドウをアライン。パーセンテージによる曖昧な幅調整は避ける。

---

## 6. Motion & Interaction
- **Tactile Transitions:**
  - ウィンドウを開く・閉じる際、およびドラッグ時の動きにはスプリング物理（stiffness: 100, damping: 20）を模した滑らかなトランジションを設定。
  - CSSの `transform` と `opacity` のみを使用し、レンダリング負荷を軽減。
- **CRT Simulation:**
  - 画面全体に薄い走査線（Scanlines）エフェクト（グラデーションをリピートしたオーバーレイ）を重ね、静止時にも微細な動き（わずかな明滅アニメーション）を適用します。
- **Hover Micro-interactions:**
  - アイコンやメニューにホバーした際、白黒が反転（Invert）する古典的なGUIエフェクトを適用。

---

## 7. Anti-Patterns (Banned)
- 絵文字（😊, 🚀 など）の使用禁止。代わりにドット絵（ビットマップ）のアートやアイコンを使用。
- `Inter` フォントの使用。
- 純粋な黒（`#000000`）の背景またはテキスト。
- ネオンや発光するシャドウ（`box-shadow: 0 0 10px #ff00ff` のようなサイバーパンク表現）。
- 曖昧な数値データ（「顧客満足度99.9%」などのAI典型表現）。
- 左右対称で平凡な「3連カード」レイアウト。
- AI生成の空虚なキャッチコピー（例: "Empower your digital journey" などの決まり文句）。
